Tecnologias de Armazenamento - Analytics

A amplitude do termo de Big Data e BI agrupa diversas soluções no vasto mundo dos dados e entre elas, se encontram as tecnologias com capacidade de armazenamento e processamento de grandes massas de dados, os quais garantem uma maior capacidade de rentenção de informações com custo reduzido. Entre elas, se encontram:

  • Data Warehouse

    • Data Warehouse é uma organização de banco de dados para nálises e BI, surgindo como um conceio acadêmico, além de possuir uma arquitetura e desenvolvimento voltado para processamento e armazenamento de grandes massas de dados, ou seja, é um tipo de banco com capacidade de organização de um grande volume de dados, responsável por criar e organizar relatórios por meio de históricos, ajudando na obtenção de insights e auxílio na tomada de deciões.

    O DW tem como objetivo centralizar os dados obtidos de diversas fontes e facilitar a consulta. Esses dados podem ser extraídos de fontes como planilhas por exemplo, possuindo formados diferentes, como SQL, CSV e até mesmo TxT. Além disso, é importante a compreensão de que os dados ficam disponpiveis apenas para leitura, impossibilitando comandos para modificação dos mesmos. Essa restrição assegura a integridade do dado armazenado, fazendo com que a segurança da plataforma seja aumentada.

    • DW x DM

      • Um Data Mart é um subconjunto de um Data Warehouse, sendo pequenas fatias que armazenam subconjuntos de dados, normalmente organizados para um departamento ou um processo de negócio. Normalmente, o DM é direcionado para uma linha de negócios ou equipé específica, onde seus dados costumam pertencer a um único departamento.

    • DW x BI

      • O BI é o processo de extração e tratamento de informações que apoiam a gestão de um negócio. Diante disso, normalmente, as vertentes de Business Intelligence utiliza de informações recolhidas em um DW (porém nem todos os DW's são usados no contexto de BI, pois nem todas as aplicações de BI exigem um DW).

    • DW x OLAP

      • O termo OLAP (Online Analytical Processing) se refere a um processo analítico em tempo real, como em seu nome. Sendo uma das ferramentas mais usadas para a exploração de um DW, possibilitando a alteração e análise de grande quantidades de dados de inpumeras perspectivas, como o Power BI.

  • Data Lake

    • Para as organizações, os Data Lakes signficam duas oportunidades: reduzir custos com tecnologias de sistemas caras e criar aplicativos inovadores para alanvacar o negócio. Porém, o conceito de DL é um pouco mais aprofundado, sendo um repositório de armazenmanto e engine para processamento de Big Data e BI. Além de fornecer um armazenamento massivo para qualquer tipo de dado, possuir grande capacidade de poder de processamento e capacidade de lidar com tarefas simultânea. Assim como, possibilitar a criação de correlações e obtenção de insights para apoiar tomadas de decisões mais eficientes.

    • DL x DW

      • Quando se ouve falar sobre um local de armazenamento de dados que uma organização deseja analisar, o termo de Data Warehouse é o primeiro a surgir. Porém, a arquitetura do DW não possui uma capacidade preparada e estruturada para atender Big Data, onde se faz a necessidade dos Data Lake. Ambos são repositórios que consolidam diversos armazenamento de dados (logo, possuem um objetivo semelhante).

      O DL armazena dados brutos em qualquer formato, sendo responsabilidade dos analistas de dados atribuir sentido à eles. Já o DW, tende a usar a noção de um único esquema para todas as necessidades de análise. Além disso, os DL são enormes e permitem um acúmulo de informações em um ritmo mais rápido, permitindo a inserção de elementos de diferentes tipos e diferentes análises e diferentemente dos DW, os DL permitem uma maior flexibilidade com as necessidades de negócio.

      Assim como, os DL permitem que a informação seja compartilhada com vários usuários e agrupada por temas e objetivos.

    No processo de implantação de um Data Lake, é extremamente importante considerar o papel da governança de dados. Devido suas caractetrísticas e capacidade de armazenamento, ocorre a possibilidade de não compreenção do tipo e formato do dado que está alocado, caso uma organização não seja realizada. Fazendo assim a necessidade de um padrão de governança para cada situação. Resumindo, a ideia do DL é ter um único reositório dentro da empresa para que todos os dados brutos sestejam disponíveis a qualquer pessoa que precise fazer uma análise sobre eles.
